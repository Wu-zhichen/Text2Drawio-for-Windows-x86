# 输出模式规范

## 1. 不可编辑位图

### PNG

- 作为科研流程图、架构图、原理图和评测框架的默认格式。
- 保留无损边缘和文字，建议使用横向高分辨率画布。
- 插入 Word 前检查实际版心尺寸下的可读性。

### JPG

- 仅用于用户明确要求 JPG、照片型插图或文件体积优先的情况。
- 使用高质量编码；若图中包含大量文字和细线，同时保留 PNG 母版。
- 不反复压缩同一 JPG。

位图中的模块、文字和箭头无法在 draw.io 中分别编辑。不得把“可在图片编辑器中裁剪”描述为“结构可编辑”。

## 2. 可编辑 draw.io

### 正确的生成关系

```text
风格参考 + 图结构规格
        ↓
ChatGPT Image 2 视觉概念稿
        ↓
draw.io 原生对象重建
        ↓
.drawio 源文件 + PNG 预览
```

ChatGPT Image 2 负责探索版式、配色和图标语言；draw.io 负责形成真正可编辑的结构。不要声称图像模型直接输出了原生 `.drawio`。

### 原生对象要求

- 模块：独立 `mxCell` vertex。
- 文本：独立或属于对应模块的可编辑 value。
- 连接线：独立 `mxCell` edge，并设置明确 source、target。
- 分组：使用 parent-child 关系或容器/泳道，不把多个模块焊成一张位图。
- 图标：优先用 draw.io 内置矢量图形；若必须使用位图图标，保持它只承担图标角色。
- 路由：使用 `edgeStyle=orthogonalEdgeStyle`，必要时设置航点。
- 分层：至少区分内容、连接线和可选参考背景；交付前移除或隐藏描摹背景。

### 模板使用

复制 `assets/drawio-scientific-template.drawio` 后再编辑。模板只提供画布、科研配色示例、卡片和正交连接线，不是最终内容。

### 验收

1. XML 能被解析，且根节点为 `mxfile`。
2. 在 draw.io 桌面应用、CLI 或官方 diagrams.net Web 应用中打开且无修复警告。
3. 至少抽查每类模块、文本和连接线能单独选择和修改。
4. 移动模块后连接线仍吸附于正确端点。
5. 导出 PNG 后无裁切、字体替换、交叉和重叠。
6. Word 中插入 PNG 预览，同时把 `.drawio` 作为独立交付物保留。
