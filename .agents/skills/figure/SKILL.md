# 自定义风格科研配图与 DOCX 更新

## 总原则

- 把源 DOCX 视为只读底稿，始终另存为新文件；不得删除或覆盖现有文件。
- 锁定全部既有正文、标题、表格文字、引用、页眉页脚和审批区。只允许替换图片，以及在用户授权时新增图片段落和图题。
- 优先采用用户放入 `style-references/user/` 的论文或图片；没有用户素材时，必须询问是否使用默认风格，不得自行跳过确认。
- 用户未指定输出类型时，必须询问选择不可编辑 PNG/JPG，还是可编辑 `.drawio`。
- 使用 ChatGPT Image 2 对每幅图单独生成或编辑。若图像工具不暴露模型选择，使用平台当前图像生成入口并如实记录，不得虚称已显式选择模型。
- 把“无交叉、无重叠、无截断、无伪造结果”设为硬验收条件。

## 目录与偏好

先读取 [preferences.yaml](config/preferences.yaml)，再检查以下目录：

- `style-references/user/`：用户放置喜欢的 PNG、JPG、JPEG、WEBP、SVG、PDF 或 `.drawio` 示例；非空时优先使用。
- `style-references/default/`：默认风格配置和默认参考资料。当前默认风格见 [style-profile.yaml](style-references/default/style-profile.yaml)。
- `assets/drawio-scientific-template.drawio`：可编辑模式的原生 draw.io 起始模板。

允许用户直接修改 `style-references/default/style-profile.yaml`，并向默认目录增添论文或图片，以形成自己的默认风格。除非用户明确要求“更新默认风格”，否则不得写入或覆盖这些默认偏好文件。

## 必用能力

1. 使用 PDF 处理能力渲染并检查参考论文中的代表性配图。
2. 使用 imagegen 能力生成或编辑视觉概念图；每次只处理一幅图。
3. 使用 draw.io 桌面应用、CLI 或官方 diagrams.net Web 应用创建、打开、校验和导出可编辑图。
4. 使用 documents 能力编辑 DOCX，并执行逐页渲染与 Word 实际显示核验。
5. 使用 `view_image` 检查每一幅最终 PNG/JPG 或 draw.io 导出预览。

## 工作流

### 1. 建立输入清单与内容锁

1. 确认源 DOCX、输出文件名、允许新增的图题以及正文不可修改的范围。
2. 复制源 DOCX并使用清晰版本后缀创建新文件；若目标已存在，换新名称，不覆盖。
3. 编辑前保存以下基线：
   - 正文和全部表格单元格中的既有段落文本及顺序；
   - 标题层级、节属性、页边距和页面尺寸；
   - 页眉、页脚、脚注、图题和参考文献文本；
   - 内联图片数量、媒体文件名及对应图题。
4. 对既有文本序列计算 SHA-256。最终文件必须复算并一致；新增图题须单独列入白名单。

### 2. 选择风格来源

1. 使用 `rg --files style-references/user` 列出用户参考素材，忽略隐藏文件和不支持的格式。
2. 若目录非空：
   - 读取所有数量合理的图片；素材较多时选取能覆盖版式、配色、箭头和图标语言的代表样本；
   - 对每篇 PDF 查找并渲染 3—6 幅架构图、流程图、方法图或评测框架图；
   - 若多份素材风格明显冲突，先让用户指定优先级；不冲突时综合提取共同视觉语法；
   - 使用用户素材，不再询问是否采用默认风格。
3. 若目录为空：停止生成并询问：“未检测到自定义风格参考，是否使用当前默认科研绘图风格？”
4. 用户同意时读取默认风格配置和默认目录素材；用户不同意时，请用户向 `style-references/user/` 添加文件或直接描述目标风格，然后重新检查。
5. 只借鉴版式、配色、线条、图标和信息密度，不复制参考图的研究内容、商标、独特插画或受保护表达。

### 3. 选择输出模式

若用户尚未明确指定，停止生成并询问选择：

- **不可编辑位图**：PNG 或 JPG。科研流程图默认推荐 PNG；仅在用户要求较小文件或照片型内容时使用 JPG。
- **可编辑图**：`.drawio`。用 GPT 图像生成视觉概念稿，再以 draw.io 原生形状、文本、分组和连接线重建；同时导出 PNG 预览供 DOCX 插入。

读取 [output-modes.md](references/output-modes.md) 并严格执行对应模式。ChatGPT Image 2 不能直接生成真正可编辑的 draw.io XML；不得把位图嵌入 `.drawio` 后宣称所有元素可编辑。

### 4. 提取风格规格

从所选参考素材中记录：画布比例、网格、模块层级、圆角、描边、阴影、箭头、图标、字体层级、注释密度、留白和颜色语义。形成一份本次任务的临时风格规格，不改写默认配置。

若采用默认风格，则使用白色背景、浅灰分区、细线圆角卡片、低饱和紫/珊瑚/青/蓝绿/金色、正交连接线、统一箭头和紧凑技术标注。

详细视觉规范与提示词见 [figure-style-and-prompts.md](references/figure-style-and-prompts.md)。生成图片前必须读取该文件，并用本次风格规格替换其中的默认风格字段。

### 5. 规划图组

根据正文已有论述建立“图—段落—用途”映射，不新增正文观点。常用图组包括：

1. 典型场景与物理/网络原理图；
2. 自然语言到结构化中间表示；
3. 能力扫描、知识约束与规则校验；
4. 参数来源、追溯与可审计生成；
5. 系统总体架构；
6. 智能体执行—反馈—有限修复闭环；
7. 任务分级与多层次评测框架；
8. 日志解析、指标计算和缺失数据处理；
9. 软件模块、运行环境与实验产物组织。

新增图片必须能由既有正文直接支撑。正文没有陈述的算法、实验结论、性能数值或系统能力不得出现在图中。

### 6. 为每幅图编写结构规格

生成前写出：

- 图的单一主旨；
- 3—6 个一级模块及每个模块的 2—5 个二级元素；
- 主流程和反馈路径；
- 每种颜色的固定语义；
- 图中文字清单；
- 禁止出现的内容。

优先使用简短英文标签降低位图文字错误率，并在 Word 图题中使用准确中文。若必须使用中文图中文字，减少字数并逐字检查。

### 7. 生成位图或 draw.io

#### 位图模式

1. 每幅图单独调用图像生成，不批量生成多个文件。
2. 将用户/默认风格规格和结构规格同时写入提示词，明确“参考视觉语言，不复制原图内容”。
3. 使用横向 3:2 或 16:9 画布、白底、高分辨率和论文级信息图观感。
4. 明确要求正交箭头使用独立通道，不得穿过卡片、图标、文字或其他箭头。
5. 保存为稳定文件名，如 `fig01_scene.png`；JPG 使用高质量导出并保留无损源图。

#### 可编辑 draw.io 模式

1. 使用 ChatGPT Image 2 生成无图号、无水印的视觉概念稿，作为布局、配色和图标语言参考。
2. 复制 `assets/drawio-scientific-template.drawio` 创建新文件，不覆盖模板。
3. 在 draw.io 中用原生 `mxCell` 顶点、文本、分组、泳道和正交边重建；每个模块、标签、图标和连接线必须可单独选择和编辑。
4. 不把整张概念位图作为最终图的主要内容；若临时放入背景层用于描摹，最终交付前移除或隐藏该层。
5. 使用 `orthogonalEdgeStyle`、明确的 source/target 和独立路由通道；反馈边从外侧绕行。
6. 在 draw.io 或官方 diagrams.net 中打开文件，检查元素可选、文字可编辑、连线吸附正常；导出同名 PNG 预览用于 Word。
7. 同时交付 `.drawio` 和 PNG；DOCX 中插入 PNG，不能直接嵌入 `.drawio` 代替预览。

评测图统一使用空值、破折号、`Pending` 或 `Not Reported`，禁止编造准确率、PDR、时延、吞吐量或提升百分比。

### 8. 逐图验收与修订

使用 `view_image` 以原始细节检查：

- 所有文字是否准确、完整、可读；
- 箭头是否穿过文字或模块，双向关系是否清楚；
- 是否存在相交连接线、悬空箭头和方向歧义；
- 模块间距、边距和对齐是否统一；
- 图例、颜色语义和字号是否与所选参考风格一致；
- 是否出现正文未支持的结论或数值。

位图发现问题时优先局部编辑；连续编辑仍出错时重新生成简化版本。draw.io 发现问题时直接调整原生对象和连接点，并重新导出预览。未通过检查的图不得插入 Word。

### 9. 安全更新 DOCX

1. 从副本操作，不修改源文件。
2. 替换既有图片时保持原关系 ID、锚点、版心宽度和图题；直接替换 `word/media/` 时核对扩展名、内容类型和关系映射。
3. 新增图片时仅在确定锚点后插入：图片居中、宽度适配版心、图片与图题同页、图题沿用源样式，并写入标题和替代文字。
4. 不重建模板，不统一重设样式，不修改无关 XML。
5. 保存后重新打开 DOCX，核对图片数、图题顺序、节属性和表格数量。

### 10. 内容与版式双重验收

1. **内容门禁**：复算既有文本 SHA-256；旧段落必须按原顺序完整出现，只允许白名单中的新图题和空图片段落。
2. **结构门禁**：核对页面尺寸、页边距、节数、表格数、内联图片数、标题层级和签章区。
3. **图片门禁**：逐图检查所选风格的一致性、文字、箭头和边距；可编辑模式额外检查 `.drawio` XML 可解析且对象为原生元素。
4. **渲染门禁**：逐页渲染 DOCX 为 PNG，制作接触表并检查全部页面。
5. **Word 门禁**：在 Microsoft Word 中确认中文字体、总页数、保存状态和关键图页；外部域更新提示默认不更新。
6. 检查无乱码、越界、截断、异常空白页、图题分离、表格断裂、图片模糊或元素重叠。
7. 保留全部源文件、中间文件、用户风格素材和默认配置，不执行删除。
